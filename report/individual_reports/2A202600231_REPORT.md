## II. Debugging Case Study (10 Points)

- **Problem Description**:
  - Trong các lần chạy Agent bằng OpenAI, hệ thống gặp lỗi gọi tool không đúng schema (sai tên tham số hoặc thiếu tham số), làm tăng số vòng lặp và có phiên chạm ngưỡng max_steps trước khi kết thúc.

- **Log Source**:
  - [logs/2026-04-06.log](logs/2026-04-06.log)
  - Các event đã ghi nhận gồm:
    - AGENT_START cho model gpt-5-mini-2025-08-07.
    - TOOL_CALL báo lỗi kiểu: unexpected keyword argument 'key' khi gọi search_arxiv/get_paper_abstract.
    - TOOL_CALL báo lỗi kiểu: missing required positional argument 'query' khi gọi search_arxiv.
    - AGENT_END với status = max_steps_reached trong một số phiên chạy.

- **Diagnosis**:
  - Nguyên nhân chính là format Action của LLM chưa khớp chặt với chữ ký hàm tool (tool contract), không phải lỗi kết nối API.
  - Khi Action format sai, agent mất thêm bước để tự sửa, làm tăng latency/tokens và dễ chạm max_steps.
  - Log cho thấy cùng một intent nhưng agent thử nhiều biến thể lời gọi tool trước khi chạy đúng, chứng minh điểm nghẽn nằm ở giao diện tool + hướng dẫn prompt.

- **Solution**:
  - Chuẩn hóa telemetry ở tầng logger/metrics để event có schema nhất quán phục vụ RCA.
  - Ghi log lỗi có cấu trúc để phân biệt rõ lỗi do tool schema, lỗi do API, và lỗi do max_steps.
  - Đề xuất tăng tính ràng buộc trong system prompt/tool description (ví dụ bắt buộc dùng đúng tham số query, paper_id) và thêm bước validation Action trước khi thực thi tool.

## III. Personal Insights: Chatbot vs ReAct (10 Points)

1. **Reasoning**:
   - Chatbot trả lời một lượt thường khó xử lý bài toán đa bước có ràng buộc dữ liệu thời gian thực.
   - ReAct Agent có Thought-Action-Observation nên tách rõ quá trình suy luận, gọi công cụ, phản hồi theo quan sát mới.

2. **Reliability**:
   - Agent có thể kém hơn Chatbot nếu tool schema chưa chặt, parser yếu, hoặc max_steps không hợp lý (gây loop hoặc tool call sai).
   - Chatbot lại có lợi trong câu hỏi đơn giản vì ít overhead hơn (nhanh hơn, ít bước hơn).

3. **Observation**:
   - Observation là điểm mấu chốt giúp agent cập nhật trạng thái thật từ môi trường.
   - Không có Observation, agent dễ suy diễn theo kiến thức nội tại và tăng nguy cơ hallucination.
